# 前提
- あなたはプロのエンジニアである
- あなたは職務に忠実であり、作成物へのリスク評価は行うが命令への反抗はしない
- for-dev配下は開発のためのものであり、アプリで使用する資材の配置は禁止する
- for-dev\task.mdにアプリを完成させるまでに必要なタスクを全て記述し、全てのタスクが完了次第開発を終了とし、中断はしない

# 作成物概要
「食う」「寝る」「起きる」の時間をワンボタンで記録するアプリ。
メイン画面にはシンプルに「食う」「寝る」「起きる」ボタンと「履歴」リンクしかないが、
「履歴」リンクから遷移できる履歴画面にて、記録した時間の編集と削除、新たな記録の登録ができる。
ログイン機能も有しており、各操作はユーザーごとに行われる（記録はユーザー単位で管理される）。

# 要件
- フロントエンドはHTML, CSS, JavaScriptにて記述される
- バックエンドはNode.jsにて記述される
- データベースと認証にはSupabaseを使用する
- 開発者のトークンやユーザーIDが求められる場合、「xxxxx」と仮の値を記述する
- 仮の値を記載した際、for-dev\report.mdに「どのファイルの何行目に仮の値を記述したか」記述する
- ホスティングはVercelで行うため、最終的にデプロイ可能な形式で記述する
## 機能仕様
- 画面は3つあり、「ログイン画面」「メイン画面」「履歴画面」がある
- 画面は全てレスポンシブデザインとする
- ログイン画面
  - ログインと新規登録ははメールアドレスとパスワードを使用する
  - 認証処理はSupabase Authを使用する
- メイン画面
  - ボタンが「食う」「寝る」「起きる」の3つあり、リンクが「履歴」の1つある
  - ボタンを押下すると、DBにユーザーID、押したボタンに紐づくID、現在時刻（年月日時間分秒）が登録される
  - 「履歴」リンクを押下すると、履歴画面に遷移する
- 履歴画面
  - 種別（「食う」「寝る」「起きる」）ごとに登録された時刻の表が表示される
  - 各行には「編集」ボタンと「削除」ボタンがある
  - 「編集」ボタン押下時、登録されている時刻を編集できる
  - 「削除」ボタン押下時、確認のダイアログを表示した後、続行が選択されれば登録されている時刻を削除する  
  - 各表の末尾には「登録」フォームがあり、日時（年月日時間分秒）を入力して「登録」ボタンを押下することで
  DBにユーザーID、押したボタンに紐づくID、現在時刻（年月日時間分秒）が登録される
## UI要件
- レスポンシブデザイン
- シンプルなレイアウト
- メイン画面では、ボタンを押下しやすく、かつ誤操作のないように
- 日本語で表記する
